# Guide for Managing and Structuring Wiki Content

This guide explains how to generate, synchronize, and structure wiki content using automated scripts and GitHub Actions workflows. The objective is to maintain clear organization and enable easy updates to files and directories.

## General Wiki Structure

The wiki relies on the `_Sidebar.md` and `Home.md` files located in the root of the `docs` directory:

1. **`Home.md`:** Contains the initial content displayed on the wiki's main page.  
2. **`_Sidebar.md`:** Defines the structure of the navigation sidebar. This file is automatically generated based on the structure of the `docs` directory, where subdirectories and their respective `.md` files are organized.

### Example Structure of the `docs/` Directory


```
docs/
├── _Sidebar.md
├── Home.md
├── 01-introduction/
│   ├── 01-Overview.md
│   ├── 02-Getting-Started.md
├── 02-main-features/
│   ├── 01-Feature1.md
│   ├── 02-Feature2.md
```


The `_Sidebar.md` file will reflect this structure by extracting titles from the names of subdirectories and `.md` files.

## Steps to Add and Structure New Wiki Content

### 1. Create New Subdirectories and `.md` Files
- **Subdirectories:**  
  Create a new subdirectory inside `docs/` and use a numeric prefix to define its order. For example:


```
docs/
├── 03-new-section/
```


- **`.md` Files:**  
  Within the subdirectory, create `.md` files with numeric prefixes to define their order in the sidebar. For example:


```
docs/
├── 03-new-section/
│   ├── 01-NewTopic.md
│   ├── 02-AnotherTopic.md
```


After adding new content, the final `docs/` directory will look like this:



```
docs/
├── _Sidebar.md
├── Home.md
├── 01-introduction/
│   ├── 01-Overview.md
│   ├── 02-Getting-Started.md
├── 02-main-features/
│   ├── 01-Feature1.md
│   ├── 02-Feature2.md
├── 03-new-section/
│   ├── 01-NewTopic.md
│   ├── 02-AnotherTopic.md
```

Adding new changes to the `docs/` directory structure will ensure that `_Sidebar.md` updates automatically after pushing to the `main` branch.

### 2. Update `Home.md` Content
Edit the `Home.md` file in the root of the `docs/` directory to include a relevant summary or introduction to the wiki content. Remember that this file contains the initial content displayed on the wiki's main page.

### 3. Update or Reorganize Existing Content
- **Renaming Directories or Files:**  
  To reorganize the order, adjust the numeric prefixes of directories or files.
- **Editing Files:**  
  Update `.md` files with new information or correct content as needed.

## Synchronizing the Wiki

The synchronization process uses a script and a GitHub Actions workflow to automate the generation and updating of the wiki content.

### Tools Used
1. **`update_sidebar.sh`:**  
   Automatically generates the `_Sidebar.md` file based on the structure of subdirectories and `.md` files.
2. **`sync-wiki` Workflow:**  
   Synchronizes changes from the main repository to the wiki repository whenever updates are made to the `main` branch.

### How to Activate Synchronization
1. Make changes to the directory structure or `.md` files.
2. Commit and push changes to the `main` branch.
3. The `sync-wiki` workflow will automatically execute to:
   - Generate a new `_Sidebar.md` file.
   - Synchronize changes with the wiki repository.
